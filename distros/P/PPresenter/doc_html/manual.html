 Portable Presenter is a package designed to give presentations. PPresenter is brought to you under a GNU licence, what makes it free to be used and extended.

The software and the manual-pages are ready to be tested and used. These 61 pages of documentation describe version PPresenter v1.17 (published 2002/08/12). It contains description of the functionality, and a few tutorials.

Please contribute to this document. You can see what kind of information is lacking. Maybe someone is willing to correct my English (it's not my native language, as you may have guessed).

  • Running a presentation.
    There are three ways to run a presentation:
    • Write a runnable perl-program using the PPresenter modules.
    • Run the same perl-program with the command present(1).
    • XML: Write the presentation in xml, and run it with the present command.

  • Structure of a show.
    Read this to start understanding how the library is structured.
    A show has
    • initiation via command-line options or the show's creation method new().
    • a list of slides
      • which have for each viewport a slide-view. Each view is written in a certain style, contain the style-elements:
        • Decoration: Solid.
        • Dynamics: default.
        • Fontset: Tk-fonts, Scalable X-fonts.
        • Formatter: Plain, Simple, Markup.
        • Template: Default.
        and, at runtime
        • for each view a program to be played.
        find, select, or change style elements.
        Debugging, designing, and creation of styles.
    • Images created with Tk::Photo or Image::Magick.
    • a list of viewports showing slides.
    • one control viewport, which shows slide notes, names of the neighbouring slides, a slide-buttons bar, and a progress bar.
    • a background menu is hidden behind the right mouse-button in a viewport with controls.
    • via a popup window, or via the controling viewport, you can control the timing of the show, selecting slides, and manage tagged slides.

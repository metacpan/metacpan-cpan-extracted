ISO3166 Countries and Subcountries from Wikipedia

The data displayed here has been generated by the module WWW::Scraper::Wikipedia::ISO3166. It is shipped in share/www.scraper.wikipedia.iso3166.sqlite and installed in a shared directory when you install the module. See the FAQ entry Where is the database? for details.

The data displayed is a combination of the data in 3 of the 4 tables in *.sqlite: countries, subcountries and subcountry_categories.

The subcountries_info table data comes from the 3rd column of ISO3166-1, and is not listed here.

The database structure is documented in the FAQ entry What is the database schema?, and is also on-line.

The Wikipedia sources for this are:

  • ISO 3166-1
  • ISO 3166-2

Also, I've used the subcountry pages linked off each country's 2-letter code entry on both of those pages.

To check your browser's rendering, scroll to the bottom and check entry 249, AX. The Å of Åland should have a ring on top of it, just as this line and Wikipedia do.

Wikipedia sorts countries by a display:none key, whereas I have chosen to sort by NFC($name), where the NFC() method comes from Unicode::Normalize. Usage of NFC() is discussed in the module's FAQ entry Why did you use Unicode::Normalize's NFC() for sorting?

A note about column headings: # is just a sequential number, while 'Db key' is the primary key in the countries table. For countries, the 'Number' column comes from ISO3166-1, while for subcountries, it is their code. 'Subcountries' is Yes or No as appropriate, for countries, while for subcounties it is their classificiation as one type of category, if that information is available. In some cases, e.g. France (for some subcountries), the category name is in a HTML heading rather than in a table, and hence is not listed.

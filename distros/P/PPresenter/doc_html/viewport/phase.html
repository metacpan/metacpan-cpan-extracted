Slide phase

Options

Portable Presenter is a package designed to give presentations. Slides may contain some dynamics, which are converted into a program on the moment the slide is shown.
One of the types of dynamic behaviour is spliting a phase into phases. Usually, each next phase of a slide adds text to the slide. You can have the number of phases still to be presented to complete the slide depicted in one of the corners of the control window.

Options

These options are supplied to the control viewport.
-showPhases => boolean
true (1, default) when you want to see how many phases are left for the current slide. This can be influenced via the background menu.

-phaseLocation => location
Put the phase symbols on the specified corner: nw, ne, se, or sw.

-phaseDirection => direction
In which direction the symbols are put. Must be 'vertical' (default) or 'horizontal'.

-phaseSymbol => string
The string describes the symbol to be used. Currently implemented are:
circle diameter color
An open circle in the specified color.
bullet diameter color
A filled circle in the specified color.
box diameter color
An open rectangle in the specified color.
square diameter color
An open rectangle in the specified color.
image file
Use the image (found in the file) as symbol.

The default diameter is 10 (pixels) and the default color is 'yellow'. The whole default is `image pinkball.gif'.

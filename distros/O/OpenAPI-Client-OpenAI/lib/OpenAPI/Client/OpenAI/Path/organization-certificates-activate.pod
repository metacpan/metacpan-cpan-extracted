=head1 NAME

OpenAPI::Client::OpenAI::Path::organization-certificates-activate - Documentation for the /organization/certificates/activate path.

=head1 OPERATIONS

=head2 POST /organization/certificates/activate

=head3 activateOrganizationCertificates

  $client->activate_organization_certificates({
      body => { ... },
  });

Activate certificates at the organization level.

You can atomically and idempotently activate up to 10 certificates at a time.

=head4 Responses

B<200 - Certificates activated successfully.>

Content-Type: application/json

Example:

    {
       "data" : [
          {
             "active" : false,
             "certificate_details" : {
                "expires_at" : 0,
                "valid_at" : 0
             },
             "created_at" : 0,
             "id" : "string",
             "name" : "string",
             "object" : "organization.certificate"
          }
       ],
       "object" : "organization.certificate.activation"
    }

=head1 SCHEMAS

=head2 OrganizationCertificate

Properties:

=over

=item * C<active> (boolean, required) - Whether the certificate is currently active at the organization level.

=item * C<certificate_details> (object, required)

=item * C<created_at> (integer, required) - The Unix timestamp (in seconds) of when the certificate was uploaded.

=item * C<id> (string, required) - The identifier, which can be referenced in API endpoints

=item * C<name> (anyOf, required) - The name of the certificate.

=item * C<object> (string, required) - The object type, which is always C<organization.certificate> .

Allowed values: organization.certificate

=back

=head2 OrganizationCertificateActivationResponse

Properties:

=over

=item * C<data> (array of OrganizationCertificate, required)

=item * C<object> (string, required) - The organization certificate activation result type.

Allowed values: organization.certificate.activation

=back

=head2 ToggleCertificatesRequest

Properties:

=over

=item * C<certificate_ids> (array of string, required)

=back

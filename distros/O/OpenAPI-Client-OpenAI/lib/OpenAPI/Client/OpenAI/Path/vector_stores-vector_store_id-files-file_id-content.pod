=head1 NAME

OpenAPI::Client::OpenAI::Path::vector_stores-vector_store_id-files-file_id-content - Documentation for the /vector_stores/{vector_store_id}/files/{file_id}/content path.

=head1 OPERATIONS

=head2 GET /vector_stores/{vector_store_id}/files/{file_id}/content

=head3 retrieveVectorStoreFileContent

  $client->retrieve_vector_store_file_content({
      body => { ... },
  });

Retrieve the parsed contents of a vector store file.

=head4 Path/query parameters

=over

=item * C<vector_store_id> (in path, required, string) - The ID of the vector store.

=item * C<file_id> (in path, required, string) - The ID of the file within the vector store.

=back

=head4 Responses

B<200 - OK>

Content-Type: application/json

Example:

    {
       "data" : [
          {
             "text" : "string",
             "type" : "string"
          }
       ],
       "has_more" : false,
       "next_page" : "string",
       "object" : "vector_store.file_content.page"
    }

=head1 SCHEMAS

=head2 VectorStoreFileContentResponse

Properties:

=over

=item * C<data> (array of object, required) - Parsed content of the file.

=item * C<has_more> (boolean, required) - Indicates if there are more content pages to fetch.

=item * C<next_page> (anyOf, required)

=item * C<object> (string, required) - The object type, which is always C<vector_store.file_content.page>

Allowed values: vector_store.file_content.page

=back

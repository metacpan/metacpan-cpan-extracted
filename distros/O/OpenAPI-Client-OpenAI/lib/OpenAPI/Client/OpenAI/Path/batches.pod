OpenAPI::Client::OpenAI::Path::batches - Documentation for the /batches path.

=head1 OPERATIONS

=head2 GET /batches

=head3 listBatches

  $client->list_batches({
      body => { ... },
  });

List your organization's batches.

=head4 Path/query parameters

=over

=item * C<after> (in query, optional, string) - A cursor for use in pagination. C<after> is an object ID that defines your
place in the list. For instance, if you make a list request and receive 100
objects, ending with obj_foo, your subsequent call can include after=obj_foo
in order to fetch the next page of the list.

=item * C<limit> (in query, optional, integer) - A limit on the number of objects to be returned. Limit can range between 1 and
100, and the default is 20.

Default: 20

=back

=head2 POST /batches

=head3 createBatch

  $client->create_batch({
      body => { ... },
  });

Creates and executes a batch from an uploaded file of requests

=head4 Request body

Content-Type: application/json

Properties:

=over

=item * C<completion_window> (string, required) - The time frame within which the batch should be processed. Currently only
C<24h> is supported.

Allowed values: 24h

=item * C<endpoint> (string, required) - The endpoint to be used for all requests in the batch. Currently
C</v1/responses> , C</v1/chat/completions> , C</v1/embeddings> ,
C</v1/completions> , C</v1/moderations> , C</v1/images/generations> ,
C</v1/images/edits> , and C</v1/videos> are supported. Note that
C</v1/embeddings> batches are also restricted to a maximum of 50,000 embedding
inputs across all requests in the batch.

Allowed values: /v1/responses, /v1/chat/completions, /v1/embeddings, /v1/completions, /v1/moderations, /v1/images/generations, /v1/images/edits, /v1/videos

=item * C<input_file_id> (string, required) - The ID of an uploaded file that contains requests for the new batch.

See L<upload file|https://platform.openai.com/docs/api-reference/files/create>
for how to upload a file.

Your input file must be formatted as a
L<JSONL file|https://platform.openai.com/docs/api-reference/batch/request-input>
, and must be uploaded with the purpose C<batch> . The file can contain up to
50,000 requests, and can be up to 200 MB in size.

=item * C<metadata> (Metadata)

See L</Metadata> below for shape.

=item * C<output_expires_after> (BatchFileExpirationAfter)

See L</BatchFileExpirationAfter> below for shape.

=back

Example:

    {
       "completion_window" : "24h",
       "endpoint" : "/v1/responses",
       "input_file_id" : "string",
       "metadata" : {},
       "output_expires_after" : {
          "anchor" : "created_at",
          "seconds" : 0
       }
    }

=head4 Responses

B<200 - Batch created successfully.>

Content-Type: application/json

=head1 SCHEMAS

=head2 Batch

Properties:

=over

=item * C<cancelled_at> (integer) - The Unix timestamp (in seconds) for when the batch was cancelled.

=item * C<cancelling_at> (integer) - The Unix timestamp (in seconds) for when the batch started cancelling.

=item * C<completed_at> (integer) - The Unix timestamp (in seconds) for when the batch was completed.

=item * C<completion_window> (string, required) - The time frame within which the batch should be processed.

=item * C<created_at> (integer, required) - The Unix timestamp (in seconds) for when the batch was created.

=item * C<endpoint> (string, required) - The OpenAI API endpoint used by the batch.

=item * C<error_file_id> (string) - The ID of the file containing the outputs of requests with errors.

=item * C<errors> (object)

=item * C<expired_at> (integer) - The Unix timestamp (in seconds) for when the batch expired.

=item * C<expires_at> (integer) - The Unix timestamp (in seconds) for when the batch will expire.

=item * C<failed_at> (integer) - The Unix timestamp (in seconds) for when the batch failed.

=item * C<finalizing_at> (integer) - The Unix timestamp (in seconds) for when the batch started finalizing.

=item * C<id> (string, required)

=item * C<in_progress_at> (integer) - The Unix timestamp (in seconds) for when the batch started processing.

=item * C<input_file_id> (string, required) - The ID of the input file for the batch.

=item * C<metadata> (Metadata)

See L</Metadata> below for shape.

=item * C<model> (string) - Model ID used to process the batch, like C<gpt-5-2025-08-07> . OpenAI offers a
wide range of models with different capabilities, performance characteristics,
and price points. Refer to the
L<model guide|https://platform.openai.com/docs/models> to browse and compare
available models.

=item * C<object> (string, required) - The object type, which is always C<batch> .

Allowed values: batch

=item * C<output_file_id> (string) - The ID of the file containing the outputs of successfully executed requests.

=item * C<request_counts> (object) - The request counts for different statuses within the batch.

=item * C<status> (string, required) - The current status of the batch.

Allowed values: validating, failed, in_progress, finalizing, completed, expired, cancelling, cancelled

=item * C<usage> (object) - Represents token usage details including input tokens, output tokens, a
breakdown of output tokens, and the total tokens used. Only populated on
batches created after September 7, 2025.

=back

=head2 BatchFileExpirationAfter

Properties:

=over

=item * C<anchor> (string, required) - Anchor timestamp after which the expiration policy applies. Supported anchors:
C<created_at> . Note that the anchor is the file creation time, not the time
the batch is created.

Allowed values: created_at

=item * C<seconds> (integer, required) - The number of seconds after the anchor time that the file will expire. Must be
between 3600 (1 hour) and 2592000 (30 days).

=back

=head2 ListBatchesResponse

Properties:

=over

=item * C<data> (array of Batch, required)

=item * C<first_id> (string)

=item * C<has_more> (boolean, required)

=item * C<last_id> (string)

=item * C<object> (string, required)

Allowed values: list

=back

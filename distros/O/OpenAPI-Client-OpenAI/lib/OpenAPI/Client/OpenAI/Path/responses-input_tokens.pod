=head1 NAME

OpenAPI::Client::OpenAI::Path::responses-input_tokens - Documentation for the /responses/input_tokens path.

=head1 OPERATIONS

=head2 POST /responses/input_tokens

=head3 Getinputtokencounts

  $client->getinputtokencounts({
      body => { ... },
  });

Returns input token counts of the request.

Returns an object with C<object> set to C<response.input_tokens> and an
C<input_tokens> count.

=head4 Responses

B<200 - Success>

Content-Type: application/json

Example:

    {
       "input_tokens" : 123,
       "object" : "response.input_tokens"
    }

=head1 SCHEMAS

=head2 TokenCountsBody

Properties:

=over

=item * C<conversation> (anyOf)

=item * C<input> (anyOf)

=item * C<instructions> (anyOf)

=item * C<model> (anyOf)

=item * C<parallel_tool_calls> (anyOf)

=item * C<previous_response_id> (anyOf)

=item * C<reasoning> (anyOf)

=item * C<text> (anyOf)

=item * C<tool_choice> (anyOf)

=item * C<tools> (anyOf)

=item * C<truncation> (TruncationEnum) - The truncation strategy to use for the model response. - C<auto> : If the
input to this Response exceeds the model's context window size, the model will
truncate the response to fit the context window by dropping items from the
beginning of the conversation. - C<disabled> (default): If the input size will
exceed the context window size for a model, the request will fail with a 400
error.

See L</TruncationEnum> below for shape.

=back
